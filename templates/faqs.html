Frequently Asked Questions

Navigate to the Upload page, select your network traffic data file in CSV format, and submit it. The CSV file should follow the same structure as the NSL-KDD train and test datasets but without the label column. The system will process the data and provide detection results.

SPYGUARD can detect multiple attack categories including Denial of Service (DoS), Probe, Remote to Local (R2L), User to Root (U2R), and Normal traffic.

After uploading your data, the system displays predictions for each network connection along with confidence scores. You can view summaries and detailed history in the Dashboard and History pages.

You can view the history of your uploads on the History page, where reports are organized by date. Select the specific date or upload entry you want, and download the corresponding detailed PDF report summarizing the detection results.

Use the Forgot Password link accessible from the login screen to securely update your password following the password requirements.

The system is trained on the NSL-KDD dataset, which is a refined version of the KDD Cup 1999 dataset, widely used for benchmarking intrusion detection systems.

Currently, the system supports uploading one file at a time. You can upload multiple files sequentially.

The detection model is periodically retrained with updated datasets to improve accuracy and adapt to new attack patterns.

Yes, all uploaded data is handled securely and used only for detection purposes. We follow best practices to protect your privacy.
